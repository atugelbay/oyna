# ТЗ: CRM-система и клиентская часть OYNA

Выжимка из полного ТЗ — только моменты, касающиеся **CRM для OYNA** и **клиентской части** (сайт, личный кабинет, сценарии пользователя).

---

## 1. Цели (релевантные CRM и клиенту)

**Клиентская часть (посетители):**
- Регистрация личного аккаунта
- Пополнение баланса времени (20/40/60 минут и т.д.)
- Вход в игры через QR-код
- Трата только игрового времени (ожидание не списывается)
- Набор очков в каждой игре
- Отображение в рейтингах (день/месяц/год/всё время)
- Получение статусов лояльности (как мили у авиакомпаний)

**CRM / бизнес:**
- Управление пользователями и их балансом
- Статистика по играм, точкам, времени
- Проведение акций и промо
- Масштабирование сети (несколько точек / городов)

---

## 2. Архитектура (клиент и облако)

**Облако (относится и к CRM, и к клиенту):**
- Центральная БД пользователей, очков, рейтингов, статусов
- **Панель администратора** (CRM)
- **Публичный сайт / личный кабинет** (клиент)
- API для всех точек

**Стек (пример):**
- Backend: Node.js (NestJS) / или Python (FastAPI)
- БД: PostgreSQL, Redis (кэш + сессии)
- **Frontend (сайт/каб/админка): React / Next.js**

---

## 3. Сущности (модель данных) — для CRM и клиента

Минимальный набор сущностей, нужных для CRM и клиентского опыта:

| Сущность | Назначение |
|----------|------------|
| **User** | id, имя/ник, телефон (обяз.), email, дата рождения, дата регистрации, статус лояльности, суммарные очки, активен/заблокирован |
| **AccountBalance** | user_id, доступное время, история операций, привязка к точке |
| **Venue** | id, название, город, адрес, часовой пояс, статусы (открыта/закрыта/на обслуживании) |
| **Transaction** | user_id, тип (покупка времени, бонус, промо, списание, корректировка), источник (касса, онлайн, админ), сумма (тенге), эквивалент в минутах, дата, venue_id |
| **LoyaltyStatus** | название (Bronze/Silver/Gold/Platinum), мин. очки, привилегии (JSON: % скидки, бонусное время и т.д.) |
| **Promo / Campaign** | id, название, описание, условия (JSON), период действия |
| **Leaderboard** | тип периода (день/месяц/год/всё время), период, room_id, позиция, user_id, total_score |
| **Team** | для турниров и рейтингов команд |
| **Clan** | для рейтингов кланов и турниров |

*(GameSession, Score, Room, GameMode и т.д. используются и в CRM отчётах, и при отображении в клиенте.)*

## 4. Клиентская часть — пользовательские сценарии

### 4.1. Регистрация
- Адаптивная мобильная версия сайта
- Поля: ФИО, ник (на моноблоке и в рейтинге), телефон (подтверждение по WhatsApp), дата рождения
- После регистрации: аккаунт + начальный статус (например, Bronze)
- В профиле: баланс (0), текущий статус, 0 очков, 0 игр

### 4.2. Пополнение времени
- **Через кассу в ТРЦ:** кассир находит пользователя по телефону/QR в админке/кассовом интерфейсе, выбирает пакет 20/40/60 мин, пробивает чек → баланс увеличивается; **клиент видит обновлённый баланс в личном кабинете**
- **Через сайт/приложение (перспектива):** авторизация → выбор пакета → онлайн-оплата → пополнение баланса

### 4.3. Вход в игру (QR и моноблок)
- У двери комнаты — моноблок с QR (привязан к комнате/игре)
- **Клиент:** сканирует QR камерой телефона → открывается сайт (если есть токен — авторизован, иначе — вход)
- Личный кабинет отправляет на backend: «начать игру в комнате X»
- Backend проверяет доступность комнаты и достаточный баланс (минимум 2 мин)
- При успехе: создаётся сессия, выдаётся session_token для игры

### 4.4. После игры
- Пользователь видит очки и место в рейтинге на сайте

### 4.5. Команды и кланы (клиент)
- В личном кабинете: создать команду или вступить по приглашению/коду
- При создании: название, приглашение по телефону/коду; проверка состава (3 взрослых или 2 взрослых + 2 ребёнка до 12 лет)
- Команда может присоединиться к клану или участвовать в открытых турнирах

### 4.6. Турниры (клиент)
- Регистрация на сайте, сбор команд, привязка к клану (например, КазНУ)
- В день турнира: капитан сканирует турнирный QR на моноблоке → проверка турнира, клана, попыток → запуск турнирной сессии

## 5. Рейтинги и лояльность (клиент + CRM)

### 5.1. Рейтинги (Leaderboards)
- День / месяц / год / all-time
- По комнатам и общий по точке; общий по сети (Overall)
- Пересчёт в реальном времени или батчами не реже раза в 5–10 минут
- **Клиент:** отображение рейтингов на сайте

### 5.2. Статусы лояльности
- Bronze / Silver / Gold / Platinum по порогам очков (настраиваются в админке)
- При достижении статуса: автообновление + уведомление (в интерфейсе, позже SMS/WhatsApp/Telegram)
- Для статусов: % скидки, бонусное время, приоритет в турнирах, доступ к ивентам

### 5.3. Призы (CRM)
- Топ-3 по городу/точке (месяц, год)
- В админке: сформировать список победителей, отметить «приз выдан», выгрузка в Excel

### 5.4. Многоуровневая система рейтингов
- По точке (venue): день/месяц/год/all-time по комнатам и общий
- Overall по всей сети — отдельный раздел на сайте
- Командные и клановые рейтинги (в т.ч. по турнирам)
- API Leaderboard: scope (user/team/clan), level (venue/city/overall/tournament), period (day/month/year/all-time/tournament_specific)

## 6. CRM: Админ-панель

Веб-интерфейс с ролями: **Администратор**, **Менеджер точки**, **Кассир/оператор**.

### 6.1. Функции админки

| Раздел | Функции |
|--------|---------|
| **Пользователи** | Поиск по телефону/имени/ID, просмотр профиля, смена статуса (админ/менеджеры), коррекция баланса времени с логом (кто изменил и почему) |
| **Транзакции** | Список операций, фильтр по точке/типу/дате, просмотр деталей |
| **Игровые сессии** | Список сессий, фильтр по дате/комнате/пользователю, статус (успешно/ошибка), диагностика при некорректном завершении |
| **Рейтинги** | Отображение по точкам, комнатам, периодам, выгрузка |
| **Настройки комнат и режимов** | Вкл/выкл игр, длительность уровня, конфиги режимов (JSON для UE) |
| **Маркетинг и акции** | Промокоды (доп. минуты/скидка), настройки статусов лояльности, просмотр активности по акциям |
| **Отчёты** | Выручка по дням/неделям/месяцам, кол-во уникальных игроков, среднее время игры, популярность комнат/игр |

---

## 7. Интерфейс моноблока (терминал в комнате) — граница клиент/сервер

- **Отображение:** название комнаты и режим, статус (свободна/идёт игра/тех.перерыв), QR для входа
- **Для оператора:** авторизация по пин-коду, ручной запуск/остановка игры, возможность «скипнуть» зависшую сессию
- Все действия только через API, без обхода сервера

---

## 8. Безопасность и доступы (CRM + клиент)

- Публичные запросы по HTTPS
- Авторизация: JWT для пользователей; отдельные токены/ключи для моноблоков и точек
- **Роли:** User, Operator, Manager, Admin
- Логирование критичных действий: коррекция баланса, смена статуса, вкл/выкл игры

---

## 9. Этапы (в части CRM и клиента)

- **Этап 1 (MVP):** регистрация, баланс, покупка пакетов через кассу, запуск/завершение сессии, сохранение очков
- **Этап 2:** рейтинги по дням/месяцам/годам, статусы лояльности, базовая админка
- **Этап 3:** промо и акции, отчёты для бизнеса

---

## Итог: что в зоне CRM, что — клиент

| Зона | Что входит |
|------|------------|
| **CRM** | Админ-панель, роли (админ/менеджер/кассир), управление пользователями и балансом, транзакции, сессии, настройки комнат/режимов, акции и промо, отчёты, рейтинги (просмотр и выгрузка), призы и выгрузка в Excel |
| **Клиент** | Публичный сайт и личный кабинет (React/Next.js), регистрация/авторизация, профиль и баланс, пополнение (касса + перспектива онлайн), вход по QR, рейтинги и статусы лояльности, команды и кланы, турниры (регистрация, сканирование QR), адаптивная мобильная версия |
